# Development Workflow

Every sprint follows this sequence:

1. Define objective.
2. Review existing documentation.
3. Update architecture if needed.
4. Approve implementation plan.
5. Implement.
6. Execute automated tests.
7. Perform manual verification.
8. Benchmark performance.
9. Update documentation.
10. Commit changes.
11. Review sprint.
12. Plan next sprint.

A sprint is not complete until all steps are finished.

---

# Definition of Done

A task is considered complete only when:

* Implementation is finished.
* Automated tests pass.
* Manual testing is successful.
* Documentation is updated.
* No critical issues remain.
* Code is committed.
* Code is pushed to GitHub.
* Sprint log is updated.

---

# Documentation Rules

* Documentation is always updated before closing a sprint.
* Important decisions must be recorded.
* No undocumented production code.
* Every document must include version information.
* Documents should be concise, accurate, and maintainable.
